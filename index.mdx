## What is Complior?

`complior` is an open-source CLI and platform for AI compliance. It scans your codebase, tests your AI systems, auto-fixes compliance issues, and generates legally required documents — all from the terminal.

## Get started

<Steps>
  <Step title="Scan your code" icon="terminal">
    Find compliance gaps with a single command. No config needed.
    ```bash
    complior scan
    ```
  </Step>
  <Step title="Generate passports" icon="id-card">
    `complior init` auto-discovers AI agents and creates passports. To manually re-discover:
    ```bash
    complior agent init          # optional — init does this automatically
    ```
  </Step>
  <Step title="Fix issues" icon="wrench">
    Auto-remediate findings — add SDK wrappers, generate documents, harden config.
    ```bash
    complior fix
    ```
  </Step>
  <Step title="Open dashboard" icon="gauge-high">
    Launch the TUI with 9 interactive pages for full compliance visibility.
    ```bash
    complior
    ```
  </Step>
</Steps>

## The compliance pipeline

Every EU AI Act obligation follows this 7-step pipeline:

| Step | Action | Command |
|------|--------|---------|
| **1. Init** | Create config + discover AI agents | `complior init` |
| **2. Scan** | Analyze code for gaps | `complior scan` |
| **3. Eval** | Test live system behavior | `complior eval --target <url>` |
| **4. Fix** | Auto-remediate issues | `complior fix` |
| **5. Document** | Generate required docs | `complior agent fria` / `policy` / `notify` |
| **6. Certify** | Produce audit package | `complior agent audit-package` |
| **7. Monitor** | Track production drift | `complior` (daemon watcher) |

<Tip>
  **Agent Passport** is the central data layer — it receives data from every step and feeds into documents, scoring, and certification.
</Tip>
