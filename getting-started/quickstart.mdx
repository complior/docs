Go from an unscanned project to a full compliance package in under 5 minutes.

<Steps>
  <Step title="Initialize project">
    Create the `.complior/` config directory with a guided onboarding wizard.

    ```bash
    complior init
    ```

    The wizard walks you through 8 steps: Theme → Project type → Trust level → Frameworks → Role → Industry → AI systems → Summary. This creates `profile.json` and **automatically discovers AI agents**, generating Agent Passports with 65–70% of fields auto-filled from AST analysis.

    <Tip>
      `complior agent init` is optional — `init` does agent discovery automatically. Use `agent init --force` to regenerate passports manually.
    </Tip>
  </Step>

  <Step title="Scan your code">
    Run 5-layer static analysis on your project. No config file needed.

    ```bash
    complior scan
    ```

    Expected output:
    ```
    Scanning project...
    ✓ L1: File presence        14 checks
    ✓ L2: Document structure    8 checks
    ✓ L3: Dependencies         12 checks
    ✓ L4: Code patterns        23 checks

    Compliance Score: 47/100 (F)
    Security Score:   62/100 (D)

    Findings: 23 total
      8 critical  │  7 high  │  5 medium  │  3 low

    Frameworks: EU AI Act, OWASP LLM Top 10, MITRE ATLAS
    ```

    {/* PLACEHOLDER: Screenshot of CLI scan output. 900×500px dark terminal */}
  </Step>

  <Step title="Auto-fix compliance issues">
    Preview and apply remediation strategies.

    ```bash
    complior fix
    ```

    ```
    15 fixes available:

    #  Fix                              Category   Score
    1  Add @complior/sdk wrapper         A: Code    +8
    2  Add disclosure system message     A: Code    +4
    3  Generate FRIA (Art.27)            B: Docs    +5
    4  Generate AI Usage Policy          B: Docs    +3
    5  Generate Worker Notification      B: Docs    +2
    6  Add input validation              A: Code    +3
    7  Add error handling                A: Code    +2
    8  Harden .gitignore                 C: Config  +1
    ...

    Predicted score: 47 → 75 (+28)

    Applied 15 fixes ✓
    New Compliance Score: 75/100 (C)
    ```

    <Note>
      Use `complior fix --dry-run` to preview without applying. Use `complior fix --ai` to have LLM fill in document placeholders.
    </Note>
  </Step>

  <Step title="Open the dashboard">
    Launch the TUI with 9 interactive pages.

    ```bash
    complior
    ```

    This starts the background daemon and opens the terminal dashboard. Navigate with hotkeys:

    | Key | Page | What it shows |
    |-----|------|---------------|
    | `D` | Dashboard | Score, deadlines, AI systems, activity log |
    | `S` | Scan | Findings by severity, per-obligation detail |
    | `F` | Fix | Fixable items with diff preview |
    | `P` | Passport | All AI systems, completeness %, obligations |
    | `O` | Obligations | 108 EU AI Act obligations with status |
    | `T` | Timeline | Visual countdown to Aug 2, 2026 |
    | `R` | Report | Export: PDF, MD, JSON, SARIF |
    | `L` | Log | Daemon events, system messages |
    | `C` | Chat | LLM compliance assistant |

    {/* PLACEHOLDER: Screenshot of TUI Dashboard page. 1200×700px dark terminal */}
  </Step>
</Steps>

## What you now have

After these 4 commands, your project contains:

```
project/
├── .complior/
│   ├── config.toml                        # Project configuration
│   ├── profile.json                       # Domain, data types, risk level
│   ├── agents/
│   │   ├── order-processor-manifest.json  # Agent Passport #1 (ed25519 signed)
│   │   └── support-bot-manifest.json      # Agent Passport #2 (ed25519 signed)
│   ├── evidence/
│   │   └── chain.json                     # Tamper-proof evidence chain
│   ├── fria/
│   │   └── fria-order-processor.md        # FRIA report (80% pre-filled)
│   ├── docs/
│   │   ├── ai-policy-hr.md               # AI Usage Policy
│   │   └── worker-notification.md         # Worker Notification (Art.26(7))
│   └── reports/
│       └── scan-report.json              # Latest scan results
```

<Check>
  Every file is cryptographically linked via the evidence chain. The entire `.complior/` directory is your audit-ready compliance package.
</Check>
